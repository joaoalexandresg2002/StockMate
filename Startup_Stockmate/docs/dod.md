# Definition of Done (DoD) — StockMate

Um item do backlog é considerado **concluído** quando todas as condições abaixo forem atendidas:

## Código

    A. Código implementado e funcionando sem erros no fluxo principal.

    B. Código legível: nomes de variáveis e funções claros.

    C. Organização: arquivos no diretório `/src` conforme padrão do projeto.

## Testes

    A. Testes manuais executados para a funcionalidade.

    B. Testes automatizados mínimos (quando aplicável) presentes em `/tests`.

## Qualidade

    A. Não existem warnings críticos ao executar o programa.

    B. Tratamento básico de erros/validações (ex.: entrada inválida, preço/quantidade negativa).

## Integração e Versionamento

    A. Commit com mensagem clara e descritiva (ex.: "feat: adicionar função X").

    B. Pull Request (caso haja) revisado por ao menos 1 membro (se aplicável).

    C. Branching: seguir fluxo `feature/*` → `dev` → `main`.

## Documentação

    A. Atualizar README.md com instruções de uso/instalação (ou docs/).

    B. Adicionar entrada no CHANGELOG ou registrar alterações relevantes no README.

## Entrega

    A. Código commitado e push para o repositório remoto.
    
    B. Versão demo testada localmente.
