# Product Backlog Prioritário – StockMate

# Visão Geral

    O Product Backlog contém todas as funcionalidades desejadas para o sistema StockMate, organizadas como histórias de usuário, com priorização e critérios de aceitação.

## US01 – Cadastrar produtos
    História:

        Como usuário, quero cadastrar produtos, para controlar meu estoque.

    Critérios de Aceitação:

        A. O sistema deve solicitar nome, quantidade e preço do produto.

        B. O sistema deve validar campos vazios ou inválidos.

        C. O produto deve ser salvo e exibido na listagem.

        D. Após cadastrar, o sistema deve exibir mensagem de confirmação.

    Prioridade: Alta

## US02 – Listar produtos

    História:
        Como usuário, quero listar os produtos, para visualizar meu estoque.

    Critérios de Aceitação:

        A. O sistema deve exibir nome, quantidade e preço de cada produto.

        B. Deve funcionar mesmo se a lista estiver vazia.

        C. Produtos devem ser exibidos em ordem alfabética.

        D. Interface clara e fácil de entender.

    Prioridade: Alta

## US03 – Remover produtos

    História:
        Como usuário, quero remover produtos, para manter o estoque atualizado.

    Critérios de Aceitação:

        A. O sistema deve verificar se o produto existe antes de remover.

        B. O sistema deve pedir confirmação antes da remoção.

        C. Após excluir, a listagem deve ser atualizada automaticamente.

    Prioridade: Média

## US04 – Atualizar produtos

    História:
        Como usuário, quero atualizar produtos, para corrigir erros de cadastro.

    Critérios de Aceitação:

        A. Usuário deve escolher qual produto deseja alterar.

        B. Nome, quantidade ou preço podem ser editados.

        C. Sistema deve validar dados antes de alterar.

        D. Produto atualizado deve aparecer corretamente na listagem.

    Prioridade: Baixa

## US05 – Salvamento automático

    História:
        Como usuário, quero que o sistema salve meus dados automaticamente, para evitar perdas.

    Critérios de Aceitação:

        A. Sistema deve salvar o estoque sempre após qualquer alteração.

        B. Salvamento deve ser rápido e transparente ao usuário.

        C. Arquivo deve ser acessível mesmo após fechar o programa.

    Prioridade: Baixa
