# Checklist de Code Review – StockMate

Antes de aprovar qualquer Pull Request, verifique:

## ✔ Qualidade do Código
- O código está limpo e legível?
- Existem comentários explicativos quando necessário?
- Funções são pequenas e objetivas?

## ✔ Nomes e Padrões
- Variáveis possuem nomes claros e descritivos?
- Foi seguido o padrão de escrita (snake_case)?

## ✔ Funcionamento
- A funcionalidade cumpre a User Story?
- Não quebra outras partes do sistema?
- Foi testada manualmente?

## ✔ Git e Versionamento
- A branch está correta? (feature → dev)
- Commits têm mensagens claras?
- Não há arquivos desnecessários incluídos?

Esse checklist ajuda a garantir que o código segue boas práticas.
