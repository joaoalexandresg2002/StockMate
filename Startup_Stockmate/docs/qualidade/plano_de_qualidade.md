# Plano de Qualidade – StockMate

Este plano define como o time garante que o sistema seja entregue com qualidade.

## 1. Verificação dos Requisitos
- Conferir se as funcionalidades implementadas correspondem às User Stories.
- Validar os critérios de aceitação.
- Revisar alterações no Product Backlog.

## 2. Verificação da Documentação
- Garantir que todas as fases foram documentadas.
- Conferir se a estrutura de diretórios está organizada.
- Manter README e docs atualizados.

## 3. Verificação da Interface
- Avaliar protótipos de baixa e alta fidelidade.
- Conferir consistência visual.
- Testar usabilidade básica.

## 4. Verificação do Código
- Analisar via checklist de code review.
- Garantir padronização e clareza.
- Conferir se não há duplicações.

## 5. Verificação Final
- Testes manuais realizados.
- Versão estável aprovada em dev antes de ir para main.

Esse plano garante organização, consistência e qualidade durante todo o projeto.
